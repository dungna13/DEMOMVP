1. Project Overview & Context
DocMind is a Source-Centric Document Intelligence application that allows users to load multiple documents, activate or deactivate them as AI context, and engage in a grounded conversation with an AI that cites its sources inline. The experience is inspired 100% by Google NotebookLM, elevated with a premium Glassmorphism aesthetic.

1.1 Core Design Philosophy
• Futuristic & Premium - the interface must feel like a next-generation research tool.
• Source-Grounded - every AI answer traces back to an activated source. No hallucination.
• Citations as First-Class Citizens - [1][2] labels are interactive, not decorative.
• Streaming-First - responses stream token-by-token, not wait-then-render.
• Glassmorphism Aesthetic - dark mode, blur, depth, soft glow on interactive elements.

1.2 Triple-Pane Architecture
The UI is divided into three permanent panes, mirroring NotebookLM's layout philosophy:

PaneNameResponsibilityLeft (280px)Source ManagerUpload, list, toggle, and delete sources. Tag display. Drag-to-reorder.Middle (flex)Source Guide & ViewerAI-generated notebook summary, suggested questions, full document reader with highlight sync.Right (380px)Grounded ChatConversational AI restricted to activated sources. toggleSource(sourceId) {

  // citations = [{ id:1, source_id:"abc", chunk_index:3, text:"..." }]
  return htmlContent.replace(/\[(\d+)\]/g, (match, num) => {
    const citation = citations[parseInt(num) - 1];

      class="citation-badge"
      data-source-id="${citation.source_id}"
      data-chunk-index="${citation.chunk_index}"
      data-excerpt="${escapeHTML(citation.text.slice(0,200))}"

  });

3.5.2 Chunker Design (chunker.py)
def chunk_text(text: str, source_id: str) -> list[Chunk]:
    sentences = sent_tokenize(text)  # NLTK
    chunks = []
    buffer, buf_tokens = [], 0
    chunk_index = 0

    for sentence in sentences:
        tokens = len(sentence.split())
        if buf_tokens + tokens > TARGET_TOKENS and buffer:
            chunks.append(Chunk(
                id=f"{source_id}_{chunk_index}",
                source_id=source_id,
                index=chunk_index,
                text=" ".join(buffer),
                token_count=buf_tokens
            ))
            chunk_index += 1
            # Keep last N sentences as overlap
            buffer = buffer[-OVERLAP_SENTENCES:]
            buf_tokens = sum(len(s.split()) for s in buffer)
        buffer.append(sentence)
        buf_tokens += tokens

    if buffer:  # flush remainder
        chunks.append(Chunk(...))
    return chunks

3.6.1 Retriever (retriever.py)
Before every chat turn, retrieve the top-K most relevant chunks from active sources using BM25 scoring:

from rank_bm25 import BM25Okapi

def retrieve_top_chunks(query: str, source_ids: list[str], top_k=8) -> list[Chunk]:
    all_chunks = load_chunks_for_sources(source_ids)
    tokenized = [c.text.lower().split() for c in all_chunks]
    bm25 = BM25Okapi(tokenized)
    scores = bm25.get_scores(query.lower().split())
    top_indices = sorted(range(len(scores)),
                         key=lambda i: scores[i], reverse=True)[:top_k]
    return [all_chunks[i] for i in top_indices]

3.6.2 Prompt Architecture (prompt_builder.py)
The system prompt enforces two rules: answer only from sources, and always cite inline.

SYSTEM_PROMPT = """
You are DocMind, an expert research assistant.
You have access to the following source passages,
each identified by a citation number [N].

STRICT RULES:
1. Answer ONLY using information from the provided passages.
2. Every factual claim MUST include an inline citation: [1], [2], etc.
3. If the answer is not in the sources, say:
   "This information is not available in the selected sources."
4. Do NOT invent, extrapolate, or add external knowledge.
5. After your prose answer, output a JSON block:

```citations
[
  { "id": 1, "source_id": "...", "chunk_index": 3,
    "text": "...verbatim passage excerpt (max 300 chars)..." },
  ...
]
```

SOURCE PASSAGES:
{context_block}
"""

3.6.3 Context Block Format
Each chunk is numbered and labeled for Gemini to reference:

def build_context_block(chunks: list[Chunk]) -> str:
    lines = []
    for i, chunk in enumerate(chunks, 1):
        source = get_source(chunk.source_id)
        lines.append(
            f"[{i}] SOURCE: {source.name} (chunk {chunk.index})\n"
            f"---\n{chunk.text}\n"
        )
    return "\n".join(lines)

3.6.4 Response Parsing (gemini_client.py)
After streaming completes, the full response is parsed to extract the citations JSON block:

import re, json

def parse_response(raw: str) -> tuple[str, list[dict]]:
    pattern = r"```citations\s*([\s\S]+?)```"
    match = re.search(pattern, raw)
    citations = []
    prose = raw
    if match:
        prose = raw[:match.start()].strip()
        try:
            citations = json.loads(match.group(1))
        except json.JSONDecodeError:
            citations = []
    return prose, citations

3.6.5 SSE Streaming Endpoint (/chat)
from fastapi.responses import StreamingResponse

@router.post("/chat")
async def chat(req: ChatRequest):
    chunks = retriever.retrieve_top_chunks(
        req.message, req.active_source_ids
    )
    context = prompt_builder.build_context_block(chunks)
    system_prompt = SYSTEM_PROMPT.format(context_block=context)

    async def event_stream():
        full_response = ""
        async for token in gemini_client.stream(
            system_prompt, req.history, req.message
        ):
            full_response += token
            yield f"data: {json.dumps({'token': token})}\n\n"

        # Parse and send citations after stream ends
        prose, citations = parse_response(full_response)
        yield f"data: {json.dumps({'citations': citations, 'done': True})}\n\n"

    return StreamingResponse(event_stream(),
        media_type="text/event-stream")

7. Aesthetic & UX Guidelines

Every design decision in DocMind must satisfy four criteria:

CriterionImplementation RuleFuturisticDeep navy backgrounds (#0a0e1a), radial gradient depth, glowing accents. Never white backgrounds in panels.Clean & MinimalMaximum information density with zero visual clutter. Icons, not labels where possible. 8px base grid.High-end / PremiumGlassmorphism on all interactive surfaces. Subtle glow on hover. Smooth 200ms transitions everywhere.Content-FocusedUI chrome is minimal. Source text and AI responses dominate the visual weight. No decorative elements.Tonal HierarchyThree text colors only: primary (#e8eaf6), secondary (#94a3b8), muted (#4a5568). No other text colors.Interaction FeedbackEvery click, toggle, upload triggers an immediate visual response. Never silent actions.Dark Mode OnlyDocMind is dark-mode-native. No light mode toggle. Deep blue is the identity.
? Key Premium Detail - Citation Badges
The [1][2] citation badges are the signature interaction of DocMind.
They must feel like a premium feature: glowing blue pill, smooth hover expansion,
popover with frosted glass background. This is the element users will screenshot and share.
Invest disproportionate design time here.
? Anti-Patterns - Never Do These
• Never use a white or light background on any panel.
• Never show a spinner without a skeleton screen underneath.
• Never allow the chat to respond without active sources - show a clear warning.
• Never use non-glass modals or alerts - everything must maintain the aesthetic.
• Never let the AI answer without citations - enforce this at the prompt level.


8. Summary
DocMind is a fully-specified, production-ready Document Intelligence platform. This implementation plan covers every layer from CSS tokens to Gemini prompt engineering. The eight phases are sequenced to deliver working vertical slices early - Phase 4 (citations) is the core value proposition and receives the most implementation time.

The system's defining technical innovations are:
• Source-centric activation model - only checked sources enter the AI context window.
• BM25 retrieval - deterministic, fast, no vector database required for v1.
• Structured citation protocol - Gemini outputs a parseable JSON citations block alongside prose.
• SSE-first streaming - every AI response streams token-by-token; no blocking waits.
• Citation-to-source sync - clicking [1] in chat scrolls and highlights in the Document Reader.

This plan is ready for immediate implementation. Phases 1-4 (frontend) and Phases 5-6 (backend) can be developed in parallel by two engineers.
